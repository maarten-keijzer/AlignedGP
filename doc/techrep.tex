\begin{abstract}
Semantic inversion of intervals makes it possible to backpropagate the hits loss metric in symbolic regression. The backpropagated intervals are used together with a constant stabbing routine to find optimal constants in a genetic programming system.

Results are mixed so far. Though the average hits with recursive stabbing are quite a bit better than standard GP, success rates are not. That sucks.
\end{abstract}

\section{Introduction}

\section{Semantic inversion using interval arithmetic}

Semantic backpropagating \cite{krawiec} use analytical inversion to transform root level targets into surrogate targets that are used in a subsequent search for a good sub-tree for that position. While this preserves the location of the target value, it does not preserve the error structure; a subtree that is a better fit to the surrogate target according to some metric need not lead to a better overall error at the root level. Residual errors will get mapped in non-linear ways, possibly leading to very large errors at the root level. Only when the new sub-tree dominates the original sub-tree on all surrogate targets can a guaranteed improvement be realized. 

Here, instead of using a squared error or related measure for judging different trees, we use interval bands. Target values are bound with one or more interval bands, with each band representing a tolerance for being off-target.  

Given a set of targets $[t_1, \ldots, t_n]$, the symbolic regression problem is reformulated to hitting targets $[(t_1 - \epsilon, t_1 + \epsilon, \ldots, (t_n - \epsilon, t_n + \epsilon) ]$, or in general hitting a set of intervals $[(l_1, u_1), \ldots, (u_n, l_n)]$ 

When using a single band, the metric reduces to Koza's original hits metric \cite{jaws1}.

With multiple equally spaced bands ($\epsilon, 2\epsilon, \ldots$), the fitness metric is roughly equivalent to $\epsilon$-lexicase selection \cite{eps-lexicase} with pre-set bands. 

The advantage of using interval inversion is that hits are maintained during semantic inversion. If a subtree hits (stabs) a surrogate target's interval, it is guaranteed that the overall tree will also hit the original target, and therefore scores a hit when inserted. The hits metric is invariant under inversion. 

However, using these surrogate targets for selecting possible sub-trees is not immediately useful. The efficiency gains when doing sub-tree selection at depth $d$ in the tree will only save $d n$ evaluations compared to a naive insertion of these same sub-trees in the overall tree directly, and evaluate it at the root level. Methods like AGX \cite{krawiec} use kd-trees to efficiently search through a library of potential subtrees to make te effort put in inversion pay off. 

\section{Computing interval inversions}

Given a node whose output must lie in a target interval $[l, u]$, interval inversion maps that requirement backward through the node's operation to produce a surrogate target for the child subtree. Each of the $n$ data points is handled independently, so the result is a vector of surrogate intervals, one per point.

\section{Stabbing}

Stabbing problems are studied in computational geometry to find the number of areas a ray, line, or general shape intersects with \cite{geometry}. 

In symbolic regression with interval (surrogate) targets, the areas of interest are the (surrogate) target intervals, and trees try to hit (stab) as many intervals as possible. When all intervals are stabbed, a 100\% hit rate is achieved.

Finding an interval that intersects with the maximum number of intervals from a set is an $O(n log n)$ operation in the number of intervals considered; i.e.; given a set of interval targets, the algorithm finds a range of constants, each constant from that range stabbing the maximum number of intervals that are achievable.

Therefore, given a subtree $T$ producing evaluations $x$ against a surrogate target $[(l_1, u_1), ..., (l_n, u_n)]$, the maximum overlapping interval algorithm gives us a range of values $c \in (l,u)$ for which $x + c$ stabs most intervals, i.e., maximizes hits. 

Given the invariance property of interval inversion on the number of hits, this makes it possible to find optimal additive constants anywhere in the tree. Instead of using unguided or gradient search to iteratively find a good constant, the optimal constant(s) can be found in $O(n log n)$ time. When this routine is used for crossover or mutation, it will ensure that a new subtree will be brought into the right range, stabbing at least one interval. 


Finding also the scale instead of just the intercept has complexity $O(n^2)$, and is therefore too expensive to run in the inner loop of a symbolic regression algorithm.

\subsection{Selecting stabbing constants}
\label{sec:selecting}

The maximum interval overlap routine returns one or more intervals. Any value within that range is a valid constant for maximizing the number of hits. When selecting a constant, it is first checked if the value 0 is part of the range. If that is the case, the value is chosen for the constant because it reduces structural complexity (x + 0 = x).

Otherwise, another check is performed to see if there are integers that are optimal. If there are such integers, an integer is drawn uniformly from the intervals. This implements a preferential bias towards integer constants. 

Finally, if there is no zero, nor integers in the range, a constant is drawn uniformly from the allowable ranges.

\subsection{Setup and notation}
Fix data points $\mathbf{x}_1,\dots,\mathbf{x}_n$ and target sets
$t_1,\dots,t_n \subseteq \mathbb{R}$. The targets may be intervals, unions of
intervals, or unions of rays.

Adjoin a symbol $\bot$ for "undefined" (e.g. division by zero) and decree

$$\bot \notin t_i \quad \text{for all } i.$$

For an expression $E$, write $E(\mathbf{x}_i) \in \mathbb{R}\cup\{\bot\}$ for its
evaluation on point $i$, and define the hit count

$$\text{hits}(E) := \bigl|\{\, i : E(\mathbf{x}_i) \in t_i \,\}\bigr|.$$

**Context function.** Let $T$ be the current tree and $p$ a node position in it.
For each point $i$, define

$$g_i^p : \mathbb{R} \to \mathbb{R}\cup\{\bot\}$$

as the map sending *the value produced at node $p$* to *the value produced at the
root*, with every other input and subtree frozen at its value for point $i$.
Concretely: evaluate $T$ at $\mathbf{x}_i$, but replace the subtree at $p$ by a
free variable $u$. The result is a fixed partial function of $u$, with
$g_i^p(u)=\bot$ wherever downstream evaluation is undefined.

Let $v_i^p$ be the actual value of the subtree at $p$ on point $i$, so that

\[g_i^p(v_i^p) = T(\mathbf{x}_i). \tag{1}\]

\paragraph{Exact preimage surrogate} Define

\[
S_i^p := (g_i^p)^{-1}(t_i) = \{\, u \in \mathbb{R} : g_i^p(u) \in t_i \,\}. \tag{2}
\]

$S_i^p$ is exactly the set of
node-$p$ outputs that make point $i$ a hit. Because $\bot \notin t_i$, every $u$
producing an undefined result is automatically excluded from $S_i^p$.

\paragraph{The stab} We stab an additive offset $c$ at node $p$: replace the subtree
$s$ at $p$ by $s + c$. The node's output on point $i$ becomes $v_i^p + c$, so
point $i$ is hit iff $v_i^p + c \in S_i^p$, i.e. iff $c \in S_i^p - v_i^p$. The
max-overlap routine returns

\[
c^\ast \in \arg\max_{c\in\mathbb{R}}
\bigl|\{\, i : c \in S_i^p - v_i^p \,\}\bigr|,
\qquad M := \text{that maximum count.} \tag{3}
\]


\paragraph{Lemma 1 — Stab count equals resulting hits}

**Statement.** Let $T'$ be $T$ with node $p$ replaced by $s + c^\ast$. Then
$\text{hits}(T') = M$.

**Proof.** In $T'$, node $p$ outputs $v_i^p + c^\ast$ on point $i$, so the root
outputs $g_i^p(v_i^p + c^\ast)$. Hence

\[
\text{hits}(T')
= \bigl|\{\, i : g_i^p(v_i^p + c^\ast)\in t_i \,\}\bigr|
\overset{(2)}{=} \bigl|\{\, i : v_i^p + c^\ast \in S_i^p \,\}\bigr|
= \bigl|\{\, i : c^\ast \in S_i^p - v_i^p \,\}\bigr|
\overset{(3)}{=} M.
\]

\paragraph{Lemma 2 — Zero is feasible}

**Statement.** $M \ge \text{hits}(T)$.

**Proof.** $c = 0$ is a candidate in (3), and

\[
\bigl|\{\, i : 0 \in S_i^p - v_i^p \,\}\bigr|
= \bigl|\{\, i : v_i^p \in S_i^p \,\}\bigr|
\overset{(2)}{=} \bigl|\{\, i : g_i^p(v_i^p)\in t_i \,\}\bigr|
\overset{(1)}{=} \text{hits}(T).
\]

Since $M$ is the maximum over all $c$, we get $M \ge \text{hits}(T)$.
$\blacksquare$

---

\paragraph{Theorem — Monotone chain}

**Statement.** Let $T_0$ be any tree and let
$T_0 \to T_1 \to \cdots \to T_m$ be a sequence in which each $T_k$ is obtained
from $T_{k-1}$ by stabbing an additive offset at one node position $p_k$, using
the exact preimage surrogate computed **against $T_{k-1}$** (A1) and a correct
stabber (A2). Then

\[ 
\text{hits}(T_m) \ge \text{hits}(T_{m-1}) \ge \cdots
\ge \text{hits}(T_1) \ge \text{hits}(T_0).
\]


**Proof.** Fix a step $k$. Apply Lemma 1 with $T=T_{k-1}$, $p=p_k$, $T'=T_k$:
this gives $\text{hits}(T_k)=M_k$. Apply Lemma 2 with the same data:
$M_k \ge \text{hits}(T_{k-1})$. Hence $\text{hits}(T_k)\ge\text{hits}(T_{k-1})$.
Chain over $k=1,\dots,m$. $\blacksquare$

The hypothesis "surrogate computed against $T_{k-1}$" is the Gauss–Seidel
condition. It is essential: it is what makes $c = 0$ reproduce $T_{k-1}$ exactly
in Lemma 2. Computing a step's surrogate against a stale version of the tree
(Jacobi) removes the feasible zero and voids the chain.


\section{Stabbing crossover and mutations}

Using the machinery of interval inversion and constant stabbing, corresponding crossover and mutation operators are defined using an {\bf recursive-stabbing} routine. It inserts the tree at a position, adds an optimal constant to put it in range, and repeats this on the path upward, drawing fresh constants along the way. Redrawing constants repositions the surrogate targets for other branches, possibly making the overall problem easier to solve. 

\subsection{The recursive-stab algorithm}

Every node carries an additive constant $c$, so a node with function $f$ evaluates as $f(\cdot) + c$. The algorithm recurses from the root of $R$ toward position $i$, propagating surrogate targets downward and updated evaluations upward, re-optimising each ancestor's additive constant on the way back. 

\begin{algorithm}
\caption{$\textsc{RecursiveStab}(N, D, i, \mathbf{x}, \mathcal{T})$}
\label{alg:aligned-insert}
\begin{algorithmic}[1]
\Require recipient node $N$, donation subtree $D$, index $i$, inputs $\mathbf{x}$,
         interval targets $\mathcal{T} = [(l_j, u_j)]_{j=1}^{n}$
\Ensure  updated node $N'$, evaluations $\mathbf{v} \in \mathbb{R}^n$

\If{$i = 1$} \Comment{insertion point reached}
  \State $\mathbf{v} \leftarrow \mathrm{eval}(D, \mathbf{x}) - D.c$
  \State $c^*, \mathbf{v} \leftarrow \textsc{MaxOverlap}(\mathcal{T} - \mathbf{v})$
  \State \Return $D.\mathrm{withConstant}(c^*)$,\; $\mathbf{v} + c^*$
\EndIf

\If{$N$ is a \textbf{UnaryNode} with function $g$}
  \State $\mathcal{T}' \leftarrow g^{-1}(\mathcal{T} - N.c)$ \Comment{interval inverse, pointwise}
  \State $N'_\mathrm{child},\, \mathbf{v}_\mathrm{child} \leftarrow \textsc{RecursiveStab}(N.\mathrm{child},\, D,\, i-1,\, \mathbf{x},\, \mathcal{T}')$
  \State $\mathbf{v} \leftarrow g(\mathbf{v}_\mathrm{child})$

\ElsIf{$N$ is a \textbf{BinaryNode} with function $\oplus$ and $i \le 1 + |N.\mathrm{left}|$}
  \State $\mathbf{v}_R \leftarrow \mathrm{eval}(N.\mathrm{right}, \mathbf{x})$
  \State $\mathcal{T}' \leftarrow \mathrm{leftInverse}(\oplus,\; \mathcal{T} - N.c,\; \mathbf{v}_R)$
  \State $N'_\mathrm{left},\, \mathbf{v}_L \leftarrow \textsc{RecursiveStab}(N.\mathrm{left},\, D,\, i-1,\, \mathbf{x},\, \mathcal{T}')$
  \State $\mathbf{v} \leftarrow \mathbf{v}_L \oplus \mathbf{v}_R$

\Else \Comment{insertion is in right subtree}
  \State $\mathbf{v}_L \leftarrow \mathrm{eval}(N.\mathrm{left}, \mathbf{x})$
  \State $\mathcal{T}' \leftarrow \mathrm{rightInverse}(\oplus,\; \mathcal{T} - N.c,\; \mathbf{v}_L)$
  \State $N'_\mathrm{right},\, \mathbf{v}_R \leftarrow \textsc{RecursiveStab}(N.\mathrm{right},\, D,\, i-1-|N.\mathrm{left}|,\, \mathbf{x},\, \mathcal{T}')$
  \State $\mathbf{v} \leftarrow \mathbf{v}_L \oplus \mathbf{v}_R$
\EndIf

\State $c^*, \mathbf{v} \leftarrow \textsc{MaxOverlap}(\mathcal{T} - \mathbf{v})$ \Comment{re-optimise this node's constant}
\State \Return $N'.\mathrm{withConstant}(c^*)$,\; $\mathbf{v} + c^*$
\end{algorithmic}
\end{algorithm}

$\textsc{MaxOverlap}$ runs the $O(n \log n)$ interval-stabbing sweep and returns the optimal constant $c^*$ (selected per Section~\ref{sec:selecting}). The total cost is $O(d \cdot n \log n)$ where $d$ is the insertion depth, dominated by $d$ calls to $\textsc{MaxOverlap}$. Nodes off the path are never re-evaluated.

\section{Methods and materials}

The previous sections laid out the main contribution of the paper. This section provides further algorithmic details and describes the experimental setup.

\paragraph{Structural complexity}
The structural complexity of a tree is computed as the number of nodes in a tree. As every node has an accompanying constant, an extra count for a non-zero constant is added to the structural complexity of a tree.

\paragraph{Complexity stratification and Lexicase selection}

In our experiments, simple Lexicase selection \cite{lexicase} is used to select parents for crossover and mutation. Size controls are implemented by stratifying the population in size buckets, and lexicase selection is done inside the bucket. This not only reduces the complexity of lexicase selection by reducing the number of candidates, but makes the comparisons size-fair, i.e., parents are only competing against parents of the same size.

Size strata are chosen at random: the first stratum uniformly, and in the case of crossover, the second parent uniformly within plus or minus 10 complexity points from the first parent.

The resulting offspring is placed in the stratum corresponding to their structural complexity. After this, to keep the strata equally occupied, the stratum with most individuals is selected. A naturally elitist inverse tournament of size 2 on hits is held to identify a member for deletion. 

\paragraph{Initialization}
To create an initial population, probabilistic tree creation \cite{luke} is used. If population size allows, every straturm is initialized with the same number of initial trees. There is no attempt to find optimal constants of these trees. THey are initialized as regular trees with random constants drawn from a Cauchy distribution by dividing two independent draws from a Gaussian (randn()/randn()).  

\paragraph{Variation}
Insert locations are chosen using Koza's 90/10 rule, and subtree crossovers are applied 80\% of the time; otherwise a size fair subtree mutation is used.

After evaluation, the \emph{hits} of the tree are computed. After this, linear scaling is performed and the \emph{hits} are re-evaluated. The maximum is used for the tree's goodness-of-fit. Note that for recursive-stabbing, this operation is superfluous, as the routine ensures that maximum hits are obtained.   

With this setup, the only difference between a regular symbolic regression system and the system under study is the recursive-stabbing routine. 

\subsection{Estimating computational effort}

Comparing different symbolic regression systems for their computational efficiency to reach a certain level of performance is not straightforward. Standard metrics such as the number of individuals processed ignore important factors such as the number of fitness cases, the number of evaluations per individual, and the complexity of the individuals. A run that keeps average population size low can run many orders faster than a run that is allowed to bloat. Counting individuals processed does not distinguish those

Therefore, many comparisons are done on wall-clock time \cite{krawiec, bosman}, but this is hardware, language, and implementation dependent. Although it is suitable for comparing alternative approaches within the same symbolic regression engine, programmed by the same developer, it does not allow for algorithmic comparison. 

A natural measure for computational effort is the number of evaluations performed. It would therefore be tempting to use the size of the population times the number of fitness cases as a measure of effort. Every new individual processed would be considered to be freshly evaluated. However, evaluations can be cached, and dramatic speedups can be achieved doing this, particularly when caching is used with algorithms that heavily share subtrees (e.g., use subtree crossover almost exclusively)  \cite{alternatives-in-caching}.

A useful observation is that under assumptions of optimal caching, only new subtrees need to be evaluated. It is furthermore useful to ignore duplication effects where the same subtree is inserted at the same location in the same tree, and assume that every subtree inserted in a tree will lead to new evaluations from the insertion point upwards to the root. 

This gives a clear set of measurements to track. 

A subtree crossover at depth $d$ will add $d n$ to the overall evaluation count, and a subtree mutation inserting an (assumed new and unique) subtree of size $s$ at depth $d$ will add $(d + s) n$ to the overall evaluation count. These mutations are thus more expensive than crossovers. Computational effort is then measured in node-evaluations, or {\bf NE} for short, and effort will be measured in millions of these node-evaluations (MNE).

Under this definition of computational effort, an aligned insert, optimizing constants, costs $d n \log n$, with subtree mutation adding and additional $s n$, thus costing $s n + d n \log n$. Finding optimal constants as studied here, costs a factor of $\log n$, and for the method to be considered succesful, it will need to outperform standard methods in a comparable computational budget.
